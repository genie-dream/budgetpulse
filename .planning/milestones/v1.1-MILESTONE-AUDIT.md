### Integration Issues (Non-Blocking)

**Issue 1: BUDG-06 / No post-onboarding currency change** (unchanged from v1.0)
`BudgetEditForm` has no currency field. `handleSaveSettings` never calls `settingsStore.setCurrency()`. Currency is set once at onboarding and cannot be changed afterward in v1.
*Severity: Medium — not a v1 blocker; currency change deferred to v2*
*Affected: BUDG-06, SETT-01*

**Issue 2: TRAN-05 / locale hardcoded in DateGroupHeader call** (new finding, low)
`transactions/page.tsx` passes `locale="en"` to `DateGroupHeader`, so `smartDateLabel`'s `toLocaleDateString()` call always uses English formatting even for Korean users. i18n strings are correctly wired by Phase 6; only the date format is affected.
*Severity: Low — core TRAN-05 grouping and display satisfied; localized date formatting not a stated acceptance criterion*
*Affected: TRAN-05 (low severity only)*
